## Lessons Learned

- **When a bug report can't be reproduced after 2-3 targeted checks, stop expanding the search and ask for the exact URL/page instead.** Guessing breadth-first burns far more time than one clarifying question.
- **Before reading code to diagnose a "shipped fix isn't working" report, check whether the local clone matches what's actually deployed** (`git status` for behind/ahead, or diff local HEAD against the deployment's commit SHA via the hosting provider's API). Stale local code produces confidently-wrong diagnoses.
- **When a small CSS/image sizing fix can't be verified in the real DOM, isolate the exact markup + CSS in a minimal static repro and test it under adversarial assumptions** (a hostile ancestor rule, a clipped container) rather than just the happy path — this caught the icon distortion bug's cause (a cascade fight with framework CSS) before shipping, and `object-fit: contain` is a more robust general answer than fighting the cascade with more specific selectors.
- **Two apps can share a project's brand name** ("arigeo" marketing site vs. "arigeo" CMS admin) — don't assume which one a vague bug report refers to; the app boundary matters more than the brand name when searching.
